PORTATEC

Vamos planejar um sistema que vai se comunicar com dispositivos de controle de acesso.
Esses dispositivos serão instalados em portas e portões.
O dispositivo vai ter um wifi aberto, e quando alguém (guest/cleaner) conectar, vai digitar um PIN de 6 números, e vai poder abrir a porta.
Quando a ação for completada ou recusada, o dispositivo vai avisar o servidor.
Além disso, no sistema principal (servidor), o dono ou alguma outra pessoa indicada, vai poder enviar um comando remotamente para abrir a porta.

A comunicação entre o servidor e os dispositivos deve ser feita usando websocket (tempo real).

No sistema vamos ter autenticação para o anfitrião (host).

Vamos organizar as coisas no servidor por entidades: Users, Places (locais que possuem dispositivos), Devices (dispositivos de acesso), e Accesses (precisa pensar em um nome alternativo), Platforms (plataformas externas que fornecem bookings), Bookings.

 - Users

São Hosts que tem acesso ao sistema. Basicamente acessa usando email e senha cadastrados (por enquanto).
Os users podem criar Devices, e Places, quem cria vira o dono. Um admin pode compartilhar o place/device com outros users.
Também pode criar Accesses (PIN de acesso), e integrar uma plataforma.

- Places

São os locais criados pelos hosts. Ele pode ter mais de um dispositivo.
O place vai ter uma página onde vai mostrar os dispositivos, onde podem serem acionados remotamente.
A permissão de acesso a essa página vai depender de quem está relacionado com esse place.

- Devices

São os dispositivos que vão interagir com quem precisa de acesso. A partir do Accesses (pin de acesso) essas pessoas vão poder abrir a porta.
Vamos ter 2 tipos de dispositivos
 - Pulse: Um pulso é enviado para o dispositivo, que dependendo de ser porta ou um portão, pode mudar o comportamento.
 - Sensor: O servidor recebe dados de mudança desse dispositivo, que pode ser aberto ou fechado.

O device tem um Access (pin) padrão, que pode ser usado no local do dispositivo para abrir a porta.

No momento eu penso em ter 2 tipos de dispositivos. Um é o portatec, que é um dispositivo customizado que vai se comunicar com websocket. E outro é um dispositivo do tipo Tuya, que usará a API do Tuya para se comunicar.

- Accesses (Pin de acesso)
Nessa entidade tem o pin de 6 números, com data de início e fim. Esse pin é relacionado a um Place, então todos os dispositivos desse lugar vão receber o mesmo PIN para ser usado. E pode ser relacionado a um booking.

- Platforms
São as plataformas que o user pode integrar, vamos acessar essas plataformas para receber os bookings delas. Vamos trabalhar inicialmente com Airbnb e Booking.com usando arquivos  do tipo iCal.

- Bookings
O próprio nome já diz o que é. Precisa ter inicio e fim, e é relacionado a plataforma, e o Place. Importante ter o nome do Guest (hospede)
